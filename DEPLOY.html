☁ DEPLOY TO THE WEB

▲ VERCEL RECOMMENDED

Zero-config deploy. vercel.json already in the repo handles routing + security headers. 

# Option A — CLI (one-liner)
$ npm install -g vercel
$ vercel --prod

# Option B — GitHub integration
# 1. Push repo to GitHub
# 2. Go to vercel.com → New Project → Import repo
# 3. Framework: Vite  |  Build: npm run build  |  Output: dist
# 4. Deploy → live URL in ~30 seconds ✓

🔼 NETLIFY

netlify.toml in the repo auto-configures build command, publish dir, headers and redirects. 

# Option A — drag & drop
$ npm run build
# Drag the ./dist folder to app.netlify.com/drop

# Option B — CLI
$ npm install -g netlify-cli
$ netlify deploy --prod --dir=dist

# Option C — GitHub integration
# Connect repo at app.netlify.com → all settings auto-detected

🐙 GITHUB PAGES

Free static hosting from GitHub. Requires one extra Vite config line. 

# 1. In vite.config.ts add:  base: '/horror-fingers/'

# 2. Build & deploy
$ npm run build
$ npx gh-pages -d dist

# Live at: https://<your-username>.github.io/horror-fingers/

🖥 RENDER

Free static site hosting with automatic HTTPS. 

# 1. Create a new "Static Site" on render.com
# 2. Connect your GitHub repo
# 3. Build Command: npm run build
# 4. Publish Directory: dist
# 5. Click Create Static Site ✓

🔒 SECURITY CHECKLIST
  • ✓ Content-Security-Policy — set in index.html meta tag and in both vercel.json + netlify.toml as HTTP headers
  • ✓ X-Frame-Options: DENY — prevents clickjacking
  • ✓ X-Content-Type-Options: nosniff — prevents MIME-type sniffing
  • ✓ Referrer-Policy: no-referrer — no referrer leakage
  • ✓ Permissions-Policy — camera, microphone, geolocation all denied
  • ✓ No dangerouslySetInnerHTML — zero XSS surface
  • ✓ No eval / Function() — ESLint rule enforced
  • ✓ TypeScript strict mode — catches implicit any, unsafe ops
  • ✓ AudioContext lazy-created — respects browser autoplay policy
  • ✓ Event listeners cleaned up — all useEffect returns remove listeners
  • ✓ No secrets in codebase — .gitignore covers .env files
🩸 TROUBLESHOOTING
?

NO SOUND ON FIRST KEY PRESS

Normal browser behaviour — AudioContext is blocked until a user gesture. The context is created lazily on the first interaction and auto-resumes if suspended.

?

WHITE SCREEN AFTER GITHUB PAGES DEPLOY

Add base: '/horror-fingers/' to vite.config.ts before building. This sets the correct asset base path for sub-directory deployments.

?

FONT NOT LOADING (offline / CSP)

The app uses Google Fonts (Creepster). If your CSP blocks external fonts, download the font, place it in public/fonts/ and update the @import in src/index.css.

?

MONSTERS STAY ON SCREEN (motion disabled)

Open the Crypt Settings panel (hold top-left corner 2 s) and re-enable Motion.
